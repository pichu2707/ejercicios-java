# Buenas prácticas y convenciones

## Nombre de variables

- Usa `camelCase`: Comienza con una minúscula, luego capitaliza cada palabra que siga, por ejemplo: `firstName`, `studentCount`.
- Usa nombre descriptivos: Ya no es solamente que sea una buena práctica si no que también puede ahorrarte mucho tiempo en intentar entender tu código.
- Evita nombres de una sola letra excepto en contadores de bucles simples.

## Nombre de constantes

- Usa `UPPER_SNAKE_CASE`: Escribe en letras mayúsculas, con palabras separadas por guiones bajos, como por ejemplo: `MAX_VALUE`, `PI_VALUE`.
- Usa las constantes par alos valores que no cambian a lo largo del programa

## Reglas generales

- Los nombres pueden contener letras, dígitos, guiones bajos y signos de dólar.
- Los nombres deben comenzar con una letra o un guión bajo `_` o un signo de dolar `$`.
- Los nombres distinguen entre mayúsculas y minúsculas (`myvariable` es diferente a `myVariable`)
- Evita usar palabras clave reservadas en Java como por ejemplo `int`, `class`...
